# 10_SEED_COMPRESSION_AND_ASSEMBLY_LAW

Seed compression law: minimal text, maximal reconstruction value.

## Principles

1. zero fluff
2. zero stale duplication
3. explicit authority surfaces
4. line split always visible
5. open gaps always explicit
6. owner-readable first, technical pointers second

## Assembly layers

1. root law/trackers/gates
2. GPT paste layer (compact)
3. Codex portability layer (bounded execution)
4. review registry layer (current evidence pointers)

## Compression boundaries

Never remove:
1. source of truth declaration
2. continuity/handoff/live split
3. current active vertex
4. current open risks
5. role split owner/GPT/Codex
6. next bounded step pointer

## Transport rule

- paste blocks stay compact
- heavy evidence stays by path pointers
- canonical review roots remain the full truth surface
