  - проверка hover-only anti-pattern;
  - проверка clipping/overflow;
  - проверка overlap/out-of-bounds;
  - скриншоты по DPI/размерам.

## Найденные проблемы (по факту ревью)

### Critical
- `All screens` / Пустые или “битые” зоны при быстром переключении страниц.
  - Симптом: страница могла оставаться прозрачной после остановки fade-анимации.
  - Причина: cleanup opacity-эффекта не выполнялся при `stop()`.
  - Статус: исправлено.

### Major
- `Sessions` / clipping текстов в статусных блоках при `1366x768` и `1280x800`.
  - Причина: длинные строки + ограниченная ширина чипов.
  - Статус: исправлено (сокращение статуса + переносы в source/telemetry зоне).
- `Sessions` / риск визуального “среза” нижней части frame при плотных размерах.
  - Причина: высокая минимальная высота preview и отсутствие прокрутки в stack-контейнере.
  - Статус: исправлено (уменьшен min-size preview + страницы обёрнуты в scroll containers).
- `Validation layer` / ложные critical по analytics CTA.
  - Причина: валидатор ожидал кнопки, тогда как на экране блоки представлены заголовками/секциями.
  - Статус: исправлено в `ui_doctor` (разделены button-expectations и label-expectations).

### Minor
- Точечные copy/spacing места остаются предметом ручного acceptance-прогона.
  - Статус: допустимо для human pass, без блокеров.

## Вывод

- Есть воспроизводимый машинный контур для повторной проверки.
