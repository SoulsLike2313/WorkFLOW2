# IMPERIUM_INQUISITION_CANON_V1

Status:
- class: `foundation_inquisition_channel`
- mutability: `immutable_by_default`
- change_authority: `EMPEROR_ONLY`
- implementation_state: `channel_active_low_resource_non_destructive`

Purpose:
- active anti-heresy / cleanup / remediation channel in parallel with evolution,
- detect drift and heresy classes early,
- execute bounded remediation in pre-authorized hygiene classes,
- escalate severe cases without auto-destruction.

Heresy definition:
- canon deviation with evidence-based impact on truth, control, or sovereignty.
- heresy is not emotional disagreement.

Mandatory heresy classes:
1. fake realtime,
2. fake completeness / false PASS,
3. immutable-mutable confusion,
4. hardcoded-live confusion,
5. multi-home uncontrolled truth,
6. owner-bypass in sovereign zones,
7. non-sovereign truth override,
8. stylization masking missing truth,
9. structural canon drift.

Operating modes:
1. `silent_heartbeat_watch` (default),
2. `event_trigger_review` (changed sectors only),
3. `deep_inquisition_session` (owner/critical trigger only).

Operational ladder:
1. detect + classify + report,
2. detect + quarantine + bounded fix for pre-authorized hygiene/remediation classes,
3. severe determination/action only where explicit Emperor/owner policy exists.

Handling law:
1. detected cases go to explicit review/holding zone,
2. each case must include root-cause reasoning and evidence pointers,
3. exterminatus is proposal-only,
4. no auto-delete of canon-bearing surfaces,
5. owner review remains sovereign.

Dashboard binding requirement:
1. canon integrity status,
2. drift level,
3. active heresy alerts,
4. unresolved review cases,
5. exterminatus candidates (proposed, not executed),
6. last verified sectors.

Boundary:
1. Inquisition is remediation/enforcement channel, not sovereign authority,
2. Inquisition cannot mutate canon by itself,
3. all irreversible actions require owner decision protocol.
4. top foundation guardianship belongs to Adeptus Custodes, not Inquisition.
