# VNEXT_PROMPT_COMPRESSION_PROFILE_V1

## Status

- status: `active`
- scope: `E:\CVVCODEX`
- mode: `vNext`
- classification: `operational_compression_profile`

## Purpose

Define the next safe compression layer for Codex prompts and replies without losing truth, validation, gap visibility, owner-decision clarity, or artifact discipline.

## Proven Permanent Shorthand

These blocks are already canonically embedded and do not need full restatement in every prompt when `vNext` mode is explicit:

1. source-of-truth and safe-mirror sovereignty baseline,
2. bundle-first and repo-law-first default,
3. one-primary-truth-bundle discipline,
4. short-first reply structure,
5. single external artifact path response contract.

## Mandatory Non-Compressible Core (Hard Floor)

Compression must never drop these fields from bounded step control:

1. active primary truth line,
2. exact active delta scope (in/out),
3. explicit verdict class,
4. explicit open risk/gap boundary,
5. explicit next canonical step,
6. owner-decision trigger when fundamental choice is unresolved.

## Prompt Compression Stage (Current)

### Enforceable now

1. `CORE + DELTA` prompting remains mandatory,
2. no broad canon retelling when entrypoint/law anchors already resolve context,
3. short-first response keeps fixed five-block chat shape,
4. details stay in bundle/reports, not chat.

### Recommended operational targets (not hard law)

1. typical bounded task prompt: `18-30 lines`,
2. heavier bounded tranche prompt: `<= 80 lines`,
3. floor profile should remain block-complete, not line-minimized.

These numeric targets are recommendations only until automated lint/enforcement exists.

## Safe Lower Bound Assessment

1. safe lower bound is **block-level**, not numeric-only,
2. further compression is acceptable only at wording level while all mandatory fields stay explicit,
3. removing mandatory fields is `UNSAFE` even if prompt becomes shorter.

## What Can Be Removed from Future Prompts

Allowed to remove by default (if canonical anchors are referenced):

1. repeated long restatements of stable governance law,
2. repeated long historical bundle narratives,
3. repeated broad anti-scope lists already covered by active law surfaces.

## What Must Stay Explicit

1. current active primary truth identity,
2. single artifact entrypoint path,
3. active step boundary and target,
4. unresolved risk/gap and owner decision triggers,
5. honest status classification (`PASS`, `PASS_FOR_STEP_SCOPE`, `OPEN_PARTIAL_DELTA`, `BLOCKED`, `FAIL`).

## Boundary

1. this profile does not remove companion dependencies from truth model,
2. this profile does not create fake hard numeric line-limit law,
3. this profile does not override truth/evidence precedence.
