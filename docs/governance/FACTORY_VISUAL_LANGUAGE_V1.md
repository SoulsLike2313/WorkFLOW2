# FACTORY_VISUAL_LANGUAGE_V1

Статус:
- version: `v1`
- layer_type: `visual_language_core`
- labels_used: `ACTIVE | RECOMMENDED | FUTURE | NOT YET IMPLEMENTED | STYLE-LAYER ONLY`

## 1) Что это

Канонический визуальный язык private factory command center.
Это не декор ради декора, а рабочая оболочка вокруг truth-linked dashboard.

## 2) Зачем

1. owner должен видеть глубину и состояние фабрики быстро;
2. экран должен поддерживать доверие, а не шум;
3. визуал должен быть готов к эволюции без потери traceability.

## 3) Эмоциональный эффект (`RECOMMENDED`)

1. premium;
2. private;
3. futuristic;
4. restrained cosmic depth;
5. calm authority.

## 4) Чем private command center отличается от noisy sci-fi

1. меньше декоративных эффектов, больше информационной плотности;
2. glow и motion работают как сигналы состояния, а не как шоу;
3. каждый акцент должен помогать читать риск/gate/статус.

## 5) Что язык не должен стать

1. child sci-fi интерфейсом;
2. перегруженной neon-анимацией;
3. скучным flat-enterprise without depth;
4. визуалом, где стиль ломает truth hierarchy.

## 6) Каноническая форма

Primary pattern: `industrial command board`.
Secondary optional layer: `anatomical/organ metaphor` как interpretive overlay, не как базовая структура.

## 7) Traceability boundary (`ACTIVE`)

1. visual layer = presentation layer;
2. truth layer = bundle/artifact/source layer;
3. визуал не имеет права подменять source claims.
