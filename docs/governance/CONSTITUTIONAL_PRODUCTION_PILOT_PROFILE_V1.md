# CONSTITUTIONAL_PRODUCTION_PILOT_PROFILE_V1

## Goal
Validate that WorkFLOW2 can execute bounded real engineering missions under Constitution V1 from admission to closure without violating constitutional discipline.

## Why Pilot Starts Now
- Constitution V1 is finalized and active.
- Mission Layer is accepted baseline.
- Mission execution proof posture is `STRONG` after targeted proof closure.
- Remaining gaps are execution hardening issues, not baseline architecture gaps.

## Pilot In-Scope Mission Types
1. creator-grade successful bounded engineering mission
2. guarded/controlled engineering mission with explicit authority and policy gates
3. intentionally blocked/denied mission
4. full closure mission path including review/certification/mirror refresh/full-check

## Pilot Out-of-Scope Mission Types
- federation/factory expansion missions
- new brain-level layer implementation missions
- uncontrolled autonomous mutation missions
- product-project feature delivery missions
- heavy framework refactors for automation

## Operational Questions Pilot Must Answer
1. Can admission discipline remain stable during real bounded missions?
2. Can guarded mission paths complete with explicit policy/authority evidence?
3. Are denied paths explicit and audit-traceable without fake completion?
4. Is closure chain coherent: completion -> review/certification -> mirror refresh -> full-check?
5. Is runtime truth fresh and discoverable enough for operator decisions?
6. Is operator friction acceptable under current lightweight constitutional enforcement?

## Pilot Execution Rule
Pilot is a bounded validation phase, not a new architecture phase. Any mission outside the approved mission set is out of pilot scope.
