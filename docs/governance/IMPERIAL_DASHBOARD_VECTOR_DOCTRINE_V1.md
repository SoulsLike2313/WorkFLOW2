## Core direction

Dashboard is system-wide imperial surface, not single-project panel.

Dashboard must project both control channels:
1. Evolution channel (improvement/candidate flow),
2. Inquisition channel (drift/heresy watch and escalation).

1. multi-project,
2. multi-node,
3. emperor-centered,
4. remote-ready,
5. role-filtered access,
6. honest trust-linked state projection,
7. evolving with system maturity.

Required high-level zones:
1. imperial overview,
2. brain / constitution / canon control core,
3. true-form matryoshka core/ring/sector shell,
4. evolution channel,
5. inquisition channel,
6. live product aquarium,
7. factory production line / assembly view,
8. memory / chronology / known state,
9. living-flow zone (event-flow bloodline with changed-sector routing).

Imperial morphology direction:
1. `cosmo nebula purple metallic` style language is canonical for owner-facing shell.
2. cognition/brain morphology is allowed only when truth readability is preserved.
3. factory/product assembly morphology must remain operational and evidence-linked.

## Aquarium vector

1. human-readable live observation,
2. smooth premium visual posture,
3. evidence-linked signals,
4. no fake realtime.
5. cognition-oriented morphology is allowed when it reveals truth.
6. living flow must show active/wait/blocked/proven posture without replacing truth-layer disclosure.

## Visibility model

1. limited nodes: bounded operational overview,
2. Emperor: full integrated overview.
