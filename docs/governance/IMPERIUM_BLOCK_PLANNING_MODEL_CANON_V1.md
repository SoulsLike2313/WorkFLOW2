# IMPERIUM_BLOCK_PLANNING_MODEL_CANON_V1

## Planning Law

For accepted blocks/fronts:

1. approved block enters Astronomican
2. block carries maximal safe prediction for later intra-block work
3. new block starts with `Conditioning + compressed Execution`
4. same block/phase continuation runs as `compressed Execution only`
5. if fronts are non-conflicting in Astronomican, parallel ultra-compressed work may be allowed
6. if conflict exists, mark honestly and block parallelization

## Authority Boundary

- block route/order/prioritization mutation: `EMPEROR_ONLY`
- Administratum may maintain metadata but cannot change direction
- Regent Sigillite monitors route integrity and unauthorized mutations
