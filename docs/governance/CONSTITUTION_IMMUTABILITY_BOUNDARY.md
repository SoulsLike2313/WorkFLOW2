## 1) Boundary Principle

Constitution is foundational law surface.
Its binding mutation is sovereign-only and must not be inferred from operational activity.

## 2) Immutable-Without-Sovereign-Mutation Set

The following are immutable without explicit sovereign constitutional mutation:
1. constitutional role as top law surface (`WORKFLOW2_CONSTITUTION_V1.md`);
2. authority split where constitutional mutation is Emperor-only;
3. creator authority != Emperor sovereignty;
4. fail-closed sovereign proof model and claim-denial boundary;
5. canonical-source precedence for constitutional interpretation.

## 3) Allowed Without Sovereign Mutation

Allowed bounded actions:
1. constitutional commentary and interpretation notes;
2. proposal drafting;
3. amendment candidate preparation;
4. contradiction/drift evidence collection.

These actions are non-binding until sovereign mutation decision is executed.

## 4) Forbidden Elevation Paths

Not valid constitutional mutation authority sources:
1. creator marker alone;
2. Primarch execution without Emperor mutation authority;
3. portable/import/safe-mirror context;
4. dossier/recommendation text without sovereign mutation decision.

## 5) Mutation Boundary Check

A constitutional change is binding only when all are true:
1. authority rank = `EMPEROR`;
2. claim class = `constitutional_mutation_claim`;
3. claim-denial gates allow the mutation claim;
4. resulting constitutional state is repo-visible and validated.
