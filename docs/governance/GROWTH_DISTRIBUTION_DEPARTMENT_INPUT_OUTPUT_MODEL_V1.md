# GROWTH_DISTRIBUTION_DEPARTMENT_INPUT_OUTPUT_MODEL_V1

Статус:
- model_version: `v1`
- department: `Growth / Distribution Department`
- maturity: `SEED`

## 1) Inputs (what department receives)

1. Product Technical Map;
2. chosen product path;
3. current product state summary;
4. owner-approved positioning hints;
5. product screenshots/assets/feature summary;
6. release target and audience hints.

## 2) Output artifacts (what department emits)

1. Product Narrative Pack;
2. Channel Starter Pack;
3. Campaign Starter Plan;
4. Product Promotion File;
5. Early Sales Readiness Note.

## 3) Input quality minimums

1. нельзя работать только от абстрактной идеи без продуктовой карты;
2. claims должны иметь source proof;
3. позиционирование без owner confirmation маркируется как draft.

## 4) Output quality minimums

1. каждый выходной артефакт должен быть owner-readable;
2. нельзя включать неподтвержденные claims;
3. в каждом артефакте должен быть section "what is prohibited".

## 5) Escalation points

1. конфликт между narrative и реальным состоянием продукта;
2. попытка завысить promises для конверсии;
3. необходимость смены целевой аудитории/рынка.

## 6) Seed boundary

1. outputs = starter-ready;
2. full-scale growth operations = `NOT YET IMPLEMENTED`.
