# IMPERIUM_ASTRONOMICAN_CANON_V1

## Purpose

Astronomican is the canonical route/vector registry for IMPERIUM block execution.

## Authority

- course command: `EMPEROR_ONLY`
- route change origin: `emperor_machine_prompt`
- mutating route/order/front priority without Emperor origin: `FORBIDDEN`

## Operational Roles

- Administratum: maintains Astronomican surfaces and block metadata
- Regent Sigillite: guards Astronomican integrity and drift watch
- other organs: read and execute within approved route only

## Required Payload

Astronomican must store:

1. approved block map
2. block order
3. non-conflict / parallel status
4. per-block prediction intent
5. compressed follow-up viability
6. current active course
