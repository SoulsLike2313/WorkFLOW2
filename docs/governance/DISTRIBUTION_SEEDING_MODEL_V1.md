# DISTRIBUTION_SEEDING_MODEL_V1

Статус:
- version: `v1`
- memory_role: `minimum_distribution_cell`
- labels_used: `ACTIVE | RECOMMENDED | FUTURE | NOT YET IMPLEMENTED`

## 1) Минимальная дистрибуционная ячейка на 1 продукт (`ACTIVE`)

1. 1 main short-video lane;
2. 1 visual/social lane;
3. 1 owned channel lane.

## 2) Что это значит на практике (`RECOMMENDED`)

1. Short-video lane:
   - короткие демонстрации ценности продукта;
   - формат TikTok/Reels/Shorts-стиля.
2. Visual/social lane:
   - карточки/карусели/короткие кейсы;
   - формат Instagram/X/похожие каналы.
3. Owned channel lane:
   - канал, который контролируется owner (например Telegram/почтовый канал/сайт-база);
   - накопление аудитории вне зависимости от алгоритмов внешних платформ.

## 3) Обязательные материалы (`RECOMMENDED`)

1. Product narrative pack;
2. Campaign starter pack;
3. Posting plan file (what/where/when/why).

## 4) Product narrative pack (`ACTIVE`)

Минимум:
1. проблема, которую решает продукт;
2. для кого он;
3. чем отличается от альтернатив;
4. proof points (демо, before/after, результаты).

## 5) Campaign starter pack (`RECOMMENDED`)

Минимум:
1. 5-10 стартовых контент-единиц;
2. CTA-варианты;
3. варианты оффера для ранних пользователей;
4. трекинг базовых сигналов (просмотр, реакция, переход, запрос).

## 6) Файл "what to post / where / when / why" (`ACTIVE`)

Должен содержать:
1. что публикуем;
2. где публикуем;
3. когда публикуем;
4. зачем публикуем (ожидаемый сигнал);
5. какой следующий шаг при положительном/отрицательном результате.

## 7) Ограничения (`RECOMMENDED`)

1. не распыляться на множество каналов в самом начале;
2. не запускать seeding без готового narrative;
3. не обещать в контенте то, чего продукт пока не умеет.

## 8) `NOT YET IMPLEMENTED`

1. fully automated multi-channel campaign orchestration;
2. сложная cross-channel attribution аналитика в реальном времени.
