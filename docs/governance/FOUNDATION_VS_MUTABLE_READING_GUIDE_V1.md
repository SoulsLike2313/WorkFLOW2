# FOUNDATION_VS_MUTABLE_READING_GUIDE_V1

Status:
- class: `foundation_classifier_law`
- mutability: `immutable_by_default`
- change_authority: `EMPEROR_ONLY`

## Purpose

Define strict split between:
1. `FOUNDATION / IMMUTABLE CANON`
2. `MUTABLE / LIVE SYSTEM`

## FOUNDATION / IMMUTABLE CANON

A surface is foundation when it defines durable law, authority, semantics, or acceptance boundaries across contours.

Foundation includes:
1. constitution and governance law,
2. sovereign/authority/rank boundaries,
3. command/delegation proof model,
4. validation/enforcement laws,
5. glossary and semantic contracts,
6. stable reading order/source precedence,
7. immutable change-authority policies.

Typical paths:
1. `docs/governance/*` (law-bearing),
2. `docs/INSTRUCTION_INDEX.md`,
3. `REPO_MAP.md`,
4. `MACHINE_CONTEXT.md`,
5. stable law contracts in `workspace_config/*`.

## MUTABLE / LIVE SYSTEM

A surface is mutable when it records current operational state, delta execution, runtime status, evidence progression, experiments, or temporary roadmaps.

Mutable includes:
1. active step bundles/reports,
2. current platform/next-step state,
3. project runtime and dashboard live state,
4. contradictions/gaps/tests in progress,
5. historical checkpoint chains.

Typical paths:
1. `docs/review_artifacts/*`,
2. `runtime/*`,
3. `projects/*`,
4. `shared_systems/*`,
5. `docs/CURRENT_PLATFORM_STATE.md`,
6. `docs/NEXT_CANONICAL_STEP.md`.

## Classification tests

Classify as `FOUNDATION` when all true:
1. cross-wave durability is required,
2. artifact governs authority/acceptance/semantics,
3. routine operational delta must not mutate it.

Classify as `MUTABLE` when any true:
1. tied to active tranche/runtime,
2. expected to evolve during normal execution,
3. evidence/history carrier rather than law-bearing canon.

## Non-mixing prohibitions

1. mutable status cannot rewrite foundation law,
2. derived runtime projection cannot be labeled source-exact law,
3. project-local success cannot be treated as system-wide constitutional mutation,
4. bundle-only narration cannot replace source-precedence canon.

## Plans / hypotheses / tests / review artifacts

1. default class: `MUTABLE`,
2. promotion to foundation only by explicit Emperor authorization,
3. historical bundles stay evidence, not automatic law.

## Packaging pressure guard

Foundation reread must begin from entrypoint + indexes, not from re-assembled bundle campaigns.
Bundle campaigns are mutable review carriers, not foundation substitutes.
