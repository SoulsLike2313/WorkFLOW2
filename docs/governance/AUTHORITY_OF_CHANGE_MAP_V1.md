# AUTHORITY_OF_CHANGE_MAP_V1

Status:
- class: `foundation_authority_map`
- mutability: `immutable_by_default`
- sovereign_rule: `foundation_change_requires_emperor_authorization`

## Purpose

Define explicit change authority by artifact class.

## A. Emperor-only (sovereign change required)

1. constitutional and governance law surfaces (`docs/governance/*` with law-bearing role),
2. source precedence and authority boundaries,
3. foundation/mutable classification law,
4. canonical semantic/glossary and ontology boundaries,
5. immutable onboarding/entrypoint maps and contracts.

## B. Operational delta allowed (bounded)

1. `docs/review_artifacts/*` for active tranche evidence,
2. `runtime/*` operational outputs,
3. project/shared-system implementation files,
4. mutable status surfaces (`CURRENT_PLATFORM_STATE`, `NEXT_CANONICAL_STEP`) with explicit evidence.

Constraint:
- operational delta cannot silently rewrite Emperor-only layer.

## C. Bundle-level reassembly allowed

1. manual-safe bundle include lists,
2. bundle reports/manifests/indexes,
3. checkpoint capsules and review packaging.

Constraint:
- packaging cannot alter source truth; only represent it.

## D. Explicit owner decision required

1. gate closures,
2. final acceptance for stage transitions,
3. sovereign boundary changes,
4. reclassification from mutable evidence into foundation law.

## E. Forbidden without sovereign authorization

1. mutate foundation law through runtime or UI surfaces,
2. claim canonical closure from derived projections,
3. erase history and replace with rewritten state,
4. present bundle-only inference as constitutional truth,
5. declare full meta-index completeness while known partial boundaries remain open.
