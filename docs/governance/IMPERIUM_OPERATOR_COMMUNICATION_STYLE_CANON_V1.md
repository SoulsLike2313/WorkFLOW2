# IMPERIUM_OPERATOR_COMMUNICATION_STYLE_CANON_V1

Status:
- class: `foundation_operator_communication_law`
- mutability: `immutable_by_default`
- change_authority: `EMPEROR_ONLY`

Rule:
- operator-facing canonical surfaces use light, human-readable, precise communication by default.

Required style:
1. short visual blocks,
2. short paragraphs,
3. warm and exact language,
4. high information density without wall-of-text,
5. no repetition padding.

Truth boundary:
1. clarity must not remove technical truth,
2. evidence-linked wording remains mandatory,
3. gaps/risks/unknowns remain explicit.

Display priority:
1. owner-readable meaning first,
2. technical disclosure second via details/manifest/pointer layers.

Naming boundary:
1. use `IMPERIUM` as canonical system name in operator-facing wording,
2. keep literal file paths/tool syntax unchanged where exact technical wording is required.
