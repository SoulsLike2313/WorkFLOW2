# MANUAL_MULTI_MACHINE_SYNC_DOCTRINE_V1

Status:
- doctrine_version: `v1`
- mode: `manual_sync_primary`

## 1) Why Manual Sync

Manual sync is retained to preserve:
1. explicit acceptance control;
2. clear source precedence;
3. rank-aware authority boundaries.

## 2) Exchange Model

1. outbound portable sessions transfer operational context.
2. return bundles transfer proposals/evidence for reintegration review.
3. no bundle transfer implies sovereign rank transfer.

Canonical root expectation remains `E:\CVVCODEX` unless explicitly documented exception.

## 3) Truth/Authority Preservation

1. Emperor node keeps canonical sovereign acceptance authority.
2. Primarch/Astartes claims must remain rank-bounded.
3. safe mirror (`WorkFLOW2`) is orientation/export only.

## 4) Mandatory Handoff Metadata

Each handoff must include:
1. node rank claim and mode claim;
2. base/end head + divergence context;
3. authority contract state;
4. claim class and requested authority scope;
5. issuer identity and signature fields (`signature_status`, `signature_assurance`, `issuer_identity_status`);
6. warrant/charter references and statuses where execution authority is claimed;
7. blocker/risk declarations.

## 5) Mandatory Reintegration Checks

1. package integrity and provenance;
2. rank/authority consistency;
3. canonical root context validity;
4. claim admissibility by rank;
5. warrant/charter lifecycle validity for authority-bearing execution claims;
6. constitutional status and denial-gate outcomes;
7. decision trace: `ACCEPT | PARTIAL_ACCEPT | REJECT | QUARANTINE`.

## 6) Anti-Drift Discipline

1. claims must bind to dated runtime evidence;
2. unresolved ambiguity must remain explicit;
3. unknown identity/signature states narrow authority, never expand it.

## 7) Rank Role Summary

1. `EMPEROR`: sovereign canonical acceptance.
2. `PRIMARCH`: creator-grade non-sovereign execution/proposal.
3. `ASTARTES`: helper/integration bounded execution, warrant/charter dependent for authority-bearing execution claims.
