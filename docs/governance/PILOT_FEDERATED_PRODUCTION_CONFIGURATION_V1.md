## 5) Pilot success criteria

1. No out-of-scope execution by Astartes.
2. Every key claim has evidence path.
3. Primarch synthesis is coherent and contradiction-aware.
4. Owner receives actionable roadmap, not generic narrative.
5. Final recommendation is clearly pass/fail/blocked.

## 6) Pilot fail signals

1. fake completion without gate evidence;
2. missing entry manifests;
3. undocumented scope expansion;
4. blocker suppression.

## 7) How to test without overbuilding

1. Use existing repo docs/contracts/scripts first.
2. Keep queue handling lightweight (file/bundle discipline).
3. Add automation only after repeated friction is observed.
