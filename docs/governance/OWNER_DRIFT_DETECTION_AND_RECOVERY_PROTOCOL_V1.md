# OWNER_DRIFT_DETECTION_AND_RECOVERY_PROTOCOL_V1

Status:
- protocol_state: `ACTIVE`
- role: `owner_support_guardrail`
- authority_level: `advisory_only`

## Purpose

Detect and recover from owner/project drift patterns without moralizing, without weakening owner agency, and without breaking canon.

## Drift Types

1. `scope_explosion` - step grows beyond bounded execution capacity.
2. `canon_drift` - current direction conflicts with fixed repo law/gates.
3. `aesthetic_over_truth` - visual/form preference starts overriding proof and safety.
4. `impulse_rebuild` - restart tendency before current lane is validated.
5. `false_completion_tolerance` - pressure to call done without evidence closure.
6. `frustration_pivot` - abrupt strategic turn caused by temporary frustration.
7. `maturity_overcompression` - declaring high maturity from early scaffolds.

## Triggers and Signals

Primary signals:
1. repeated scope expansion requests within one execution lane,
2. requests to skip gates/evidence chain,
3. preference for rewrite while critical baseline defects remain unresolved,
4. completion language not backed by artifacts/proof paths,
5. conflicting directives in short interval without documented rationale.

## Temporary Impulse vs Real New Course

Treat as temporary impulse if:
1. appears once,
2. lacks repo-visible rationale,
3. conflicts with current accepted lane,
4. not repeated after one checkpoint cycle.

Treat as real new course candidate if:
1. repeated consistently across checkpoints,
2. owner states explicit strategic intent,
3. impact/risk is documented repo-visible,
4. canonical gates for change are followed.

## Recovery Protocol (Mandatory Sequence)

1. Mirror what is happening.
2. Name the concrete risk.
3. Tie back to canon and current accepted lane.
4. Offer 2-3 viable paths (bounded and explicit).
5. Recommend strongest next step with rationale.

## Intervention Tone Rules

1. Respectful and direct.
2. No moralizing, no patronizing.
3. Strong on constraints, calm on language.
4. Protect owner agency: intervention suggests, does not confiscate control.

## Anti-Overfire Controls

1. Do not escalate on every minor deviation.
2. Require at least one concrete trigger before intervention.
3. Use soft reminder first, hard block only when governance/evidence is at risk.

## Boundaries

1. Protocol cannot override governance law.
2. Protocol cannot mutate kernel.
3. Protocol cannot claim psychological diagnosis.
4. Protocol cannot replace owner final authority on approved gates.
