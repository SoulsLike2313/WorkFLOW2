## Purpose

`Worldheart Core` is the portable sovereign creator module target for future activation.
Its purpose is to keep the living center of factory authority portable, recoverable, and creator-grade without binding identity to one fixed machine.

External owner-facing name:
- `Worldheart Core`

Internal bootstrap codename:
- `First Ember`

## Why This Exists

1. Preserve sovereign creator continuity across hardware changes.
2. Keep canonical repo authority recoverable under failure conditions.
3. Prevent confusion between safe mirror visibility and sovereign control.
4. Enable planned relocation without losing governance discipline.

## Core Principles

1. Portable: authority substrate can move between machines.
2. Sovereign: source authority remains local-canonical, not remote-mirror-derived.
3. Creator-grade: acceptance and canonical mutation remain creator-governed.
4. Identity is not hardware-bound: device can change, canonical law does not.
5. Relocation survival: move should not reset governance memory or review discipline.
6. Safe mirror is support surface only, never sovereign equivalent.

## Target Architecture Layers

1. Portable sovereign SSD/NVMe core:
   - canonical repo + governance stack + authority surfaces + execution context.
2. Periodic HDD snapshot backups:
   - scheduled offline snapshots for rollback/recovery.
3. Safe remote resurrection/support layer:
   - safe mirror and safe bundles for continuity assistance, not sovereign authority replacement.

## Authority Law

1. Creator sovereignty is guarded, not "always unlocked."
2. Physical possession of a portable core is not sufficient by itself for unrestricted canonical authority.
3. Sovereign acceptance still requires conscious creator unlock and canonical gate discipline.
4. Safe coordination surfaces may assist, but cannot grant sovereign rights.

## Backup and Snapshot Discipline (Concept Draft)

1. Snapshot cadence:
   - baseline periodic full snapshot (for example: weekly),
   - lighter operational checkpoint snapshots between full snapshots.
2. Separation:
   - live creator core stays distinct from backup strata.
3. Integrity:
   - snapshot metadata must carry timestamp, head reference, and recovery notes.
4. Recovery-ready bundle posture:
   - latest canonical review/index surfaces should remain exportable as safe evidence.

## Recovery Posture (Draft)

If core drive fails, recovery should combine:
1. latest valid backup snapshots,
2. safe mirror references,
3. repository law/governance anchors,
4. review/bundle truth surfaces required to re-establish trustworthy state.

Recovery must not bypass canonical source precedence.
