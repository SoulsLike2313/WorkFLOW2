# CANONICAL_NODE_ROOT_POLICY_V1

Status:
- policy_version: `v1`
- scope: `canonical workspace root and path-drift discipline`

## 1) Canonical Node Root

Canonical node workspace root is:
- `E:\CVVCODEX`

This root is the authoritative operational base for:
1. node-rank detection;
2. authority checks;
3. runtime truth generation;
4. constitutional/admission checks;
5. reintegration control decisions.

## 2) Safe Mirror Role

`WorkFLOW2` is:
1. safe mirror;
2. browser-facing orientation surface;
3. controlled export/handoff surface.

`WorkFLOW2` is not:
1. sovereign source of truth;
2. sovereign rank proof surface;
3. replacement for canonical local workspace root.

## 3) Canonical Path Assumptions

Expected path assumptions on Emperor / Primarch / Astartes nodes:

1. workspace root path resolves to `E:\CVVCODEX`;
2. local governance/runtime validation is executed from this root;
3. external authority/proof paths may exist outside repo root, but detection is anchored to canonical root context.

## 4) Path Drift Definition

Path drift means:
1. active workspace root is not `E:\CVVCODEX`;
2. canonical checks are executed from a different root without explicit documented exception;
3. node-rank/authority claims are made from mismatched root context.

## 5) Path Drift Impact

Path drift must reduce confidence and permissions:

1. rank confidence narrows (no Emperor elevation);
2. trust/admission confidence narrows or blocks;
3. sovereign claims are denied until canonical root context is restored.

Fail-closed rule:
- root ambiguity never expands authority.

## 6) Exceptions

Allowed only when explicitly documented in repo-visible policy artifact:

1. exception must name reason, scope, and expiration/closure condition;
2. exception must define reduced-privilege behavior while exception is active;
3. no exception may auto-grant Emperor sovereignty.

Not allowed:
1. implicit root substitution by convenience;
2. treating safe mirror as sovereign root;
3. narrative root override without machine evidence.

## 7) Operational Rule

When in doubt:
1. treat root context as non-sovereign;
2. degrade to narrower rank/claim scope;
3. require explicit canonical-root revalidation.
