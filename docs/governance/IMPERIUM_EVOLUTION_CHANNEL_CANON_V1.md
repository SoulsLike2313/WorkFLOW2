# IMPERIUM_EVOLUTION_CHANNEL_CANON_V1

Status:
- class: `foundation_evolution_channel`
- mutability: `immutable_by_default`
- change_authority: `EMPEROR_ONLY`
- implementation_state: `channel_active_low_resource_not_full_autonomy`

Purpose:
- keep IMPERIUM evolution as a controlled low-cost channel,
- produce transferable improvements without scope explosion or canon drift.

Core operating law:
1. evolution reads canonical control surfaces, not arbitrary filesystem mass,
2. evolution runs in `quiet_heartbeat` by default,
3. `event_trigger` mode runs only for changed sectors,
4. `deep_review` mode requires explicit owner authorization,
5. evolution proposes candidates; owner accepts canon-impacting gains.

Read scope (default pointers):
1. current work ladder and mutable trackers,
2. storage/integrity health surfaces,
3. dashboard adapters and live semantic surfaces,
4. contradiction lanes and checkpoint/review artifacts,
5. product/runtime signals where already implemented.

Required surfaces:
1. system stage map (`current_stage` -> `target_stage`),
2. self-question bank per system line,
3. insight candidate queue,
4. owner approval queue,
5. accepted transferable gains registry,
6. resource mode disclosure (`quiet_heartbeat` / `event_trigger` / `deep_review`).

Resource discipline:
1. pointer/manifest based reads first,
2. no default full-system rescans,
3. token/limit economy is mandatory,
4. expensive scans only by explicit trigger.

Boundary:
1. evolution never rewrites immutable canon directly,
2. evolution outputs recommendations and bounded deltas only,
3. completion claims remain evidence-gated.
