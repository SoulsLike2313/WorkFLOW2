# FACTORY_STRATEGIC_ROADMAP_V1

Статус:
- version: `v1`
- memory_role: `strategic_stage_map`
- labels_used: `PROVEN | ACTIVE | MAPPED | FROZEN | NOT YET IMPLEMENTED`

## 1) Current frontier (`ACTIVE`)

1. Single-machine product hardening на машине Emperor.
2. Owner-steered gate decisions по TikTok Agent (A/B/C).
3. Bundle truth -> owner-visual parity (Local-Only Observation Layer V1).
4. Базовая дисциплина learning capture без hidden self-learning.

## 2) Next frontier (`MAPPED`)

1. Запуск Wave execution по выбранному варианту A/B/C.
2. Формализация validated pattern library на реальных wave-кейсах.
3. Укрепление department handoff протоколов между Analytics -> Engineering -> Verification.
4. Минимальная автоматизация повторяемых шагов (только после доказанной стабильности).

## 3) Medium horizon (`MAPPED`)

1. Переход от 1 продукта к устойчивому потоку похожих продуктов.
2. Расширение owner factory window до multi-product lane view.
3. Укрепление contradiction closure и before/after delta discipline.
4. Нормализация release playbooks по классам продуктов.

## 4) Long horizon (`MAPPED`)

1. Controlled multi-node federation pilot (после single-machine доказательств).
2. Расширение Primarch capacity и department specialization.
3. Throughput scaling с сохранением anti-slop и owner-control.

## 5) Frozen distant horizon (`FROZEN`)

1. Массовая high-throughput фабрика “many apps/day”.
2. Агрессивная automation-first оркестрация.
3. Широкое распределенное исполнение.

Эти направления сохраняются в памяти, но не открываются до закрытия unlock-условий.

## 6) Что делаем сейчас vs что храним в памяти

`ACTIVE` сейчас:
1. TikTok Agent gate session + hardening path.
2. Local-only observation layer bootstrap.

`MAPPED` потом:
1. Многопродуктовый конвейер и department runtime depth.

`FROZEN`:
1. Дальние масштабные программы без текущего unlock.

## 7) Анти-размытие стратегии

1. Нельзя перепрыгивать в frozen программы без closure текущего frontier.
2. Нельзя подменять stage goals “красивыми концепциями без evidence”.
3. Нельзя объявлять throughput-готовность без validated learning базы.
