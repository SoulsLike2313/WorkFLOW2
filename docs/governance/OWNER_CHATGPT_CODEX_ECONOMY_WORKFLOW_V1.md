# OWNER_CHATGPT_CODEX_ECONOMY_WORKFLOW_V1

## Status

- status: `active`
- scope: `owner + Codex + ChatGPT collaboration loop`

## Preferred Economy Workflow

1. owner gives delta-level ask,
2. Codex reads repo CORE + active primary bundle,
3. Codex executes bounded delta,
4. Codex sends short first reply with exact paths,
5. ChatGPT reviews from one declared primary bundle,
6. next step starts from latest capsule + primary bundle + new delta.

## Codex Economy Profile

Codex must:

1. avoid reprinting stable canon in prompts/replies,
2. keep first reply short and path-exact,
3. route details to bundle/report artifacts,
4. preserve one-primary-bundle discipline,
5. keep validation/evidence explicit.

Codex must not:

1. trade truth for brevity,
2. hide gaps to save tokens,
3. create competing primary bundle channels.

## ChatGPT Review Economy Profile

ChatGPT review should:

1. start from declared primary bundle,
2. treat auxiliary bundles as secondary,
3. avoid requesting broad re-narration when capsule + bundle already cover context,
4. preserve contradiction/gap honesty.

## Owner Clarity Rule

Economy mode must not degrade owner clarity.
Owner-visible summaries must remain actionable and exact-path traceable.
