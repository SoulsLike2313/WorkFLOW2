## Миссия

Формировать release-shape, готовить controlled handoff и обеспечивать прозрачную упаковку результата без ложного релизного оптимизма.

## Вход

1. verification-approved candidate bundles;
2. release target constraints;
3. integration contracts and handoff policies.

## Выход

1. release candidate bundle;
2. handoff bundle;
3. release readiness note.

## Что обязан выпускать (seed minimum)

1. charter;
2. intake/output/blocker contracts;
3. completion note;
4. status card;
5. future evolution note.

## Чего делать не может

1. объявлять sovereign acceptance;
2. публиковать продукт без owner release gate;
3. скрывать distribution risks.

## Кто руководит

1. Primarch-level release lead.

## Нужен ли Primarch

Да, обязателен.

## Какие bundle types должен понимать

1. Release Bundle;
2. Handoff Bundle;
3. Blocker/Incident Bundle;
4. Release Verdict Bundle.

## Будущие возможности (`MAPPED`)

1. standardized release tracks by product class;
2. staged distribution ladders;
3. automated release evidence assembly (bounded).
