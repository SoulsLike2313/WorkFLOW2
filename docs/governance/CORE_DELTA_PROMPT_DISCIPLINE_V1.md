# CORE_DELTA_PROMPT_DISCIPLINE_V1

## Status

- status: `active`
- scope: `E:\CVVCODEX`

## Goal

Move prompts from repeated full-context narration to stable `CORE + DELTA` discipline.

## Model

### CORE (stable, repo-resident)

CORE is already stored in canonical repo and should be referenced, not reprinted:

1. repo law and governance stack,
2. output/bundle law,
3. character/evolution law,
4. stable naming and architecture surfaces.

### DELTA (active, step-specific)

Each new execution prompt should carry only:

1. one active front,
2. one primary truth bundle line,
3. one exact open remainder,
4. one exact next move and scope boundary.

## Canonical Prompt Shape (Default)

Every new execution prompt should stay compact and include only:

1. `launch law reference`,
2. `exact sources to read`,
3. `exact active bundle + companion`,
4. `exact scope (in/out)`,
5. `hard pass/fail`,
6. `runtime info` (only if relevant).

## vNext Permanent Shorthand

If `vNext` mode is explicit, prompts may use compact shorthand and skip broad canon retelling.
Shorthand is valid only when these are still explicit:

1. active primary truth line,
2. exact bounded delta target,
3. exact in/out scope,
4. hard pass/fail boundary,
5. open risk or owner-decision trigger.

## Compression Lower-Bound Rule

`CORE + DELTA` can be compressed further only by wording reduction.
Dropping any mandatory block is outside canon even if prompt is shorter.

Numeric line targets may be used as recommendation, but they are not hard law without lint/enforcement chain.

## Forbidden Prompt Waste

Default forbidden patterns:

1. full repeat of long canon text when repo anchors already exist,
2. long system retelling across steps without delta relevance,
3. multiple competing bundle channels in one active step,
4. replacing exact scope with broad generic instructions.

## Practical Effect

This discipline reduces token spend on repetition while increasing controllability:

1. less prompt noise,
2. faster read-in,
3. stronger bounded execution.
