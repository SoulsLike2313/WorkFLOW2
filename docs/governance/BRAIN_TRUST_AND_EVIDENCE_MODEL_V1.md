## 3) Evidence Chain

Canonical trust chain:
1. law/policy defines allowed behavior;
2. contracts define machine-usable rules;
3. validators check rule conformance;
4. runtime surfaces expose current state;
5. review artifacts consolidate findings and gaps;
6. claims are permitted only within observed evidence envelope.

## 4) Confidence Marking Discipline

Mandatory claim markings:
1. `OBSERVED`: directly supported by repo-visible evidence or command output.
2. `INFERRED`: logical conclusion from observed evidence.
3. `CLAIMED-BY-DOC`: documentary statement not independently verified in current run.
4. `NOT-PROVEN`: missing or insufficient evidence.

Hard rule:
1. `INFERRED` never upgrades a `NOT-PROVEN` claim to `OBSERVED`.

## 5) Trust Classes

### T0 - Untrusted
1. core evidence missing or contradictory.
2. critical gates failing with unresolved blockers.

### T1 - Constrained
1. core governance exists;
2. partial gaps block strong claim;
3. only bounded non-sovereign outputs should be trusted.

### T2 - Operationally Trusted
1. core governing chain is coherent and machine-backed;
2. major drift/contradiction controls pass;
3. remaining gaps are explicit and narrow.

### T3 - Strong-Governing Candidate
1. T2 plus closed critical hardening fronts;
2. independent witness checkpoints pass;
3. no unresolved critical overclaim risk.

## 6) Anti-Overclaim Rules

Forbidden claims:
1. “fully hardened brain v1” while partial blocks remain unresolved.
2. sovereign authority transfer through mirror/bundle/import context.
3. stale runtime snapshots presented as current truth.
4. documentary-only semantics presented as full machine enforcement.

## 7) Human Signoff Boundaries

Human/sovereign signoff is required for:

2. canonical acceptance closure claims;
3. unresolved contradiction overrides;
4. authority-sensitive boundary changes.

4. gaps are explicitly surfaced (`NOT-PROVEN`) rather than hidden.
5. output discipline enforces evidence-rich bundle delivery for large tasks.
