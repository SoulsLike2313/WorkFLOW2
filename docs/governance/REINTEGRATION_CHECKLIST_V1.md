# REINTEGRATION_CHECKLIST_V1

Status:
- checklist_version: `v1`
- scope: `canonical reintegration gate for return bundles`

## 1) Intake Identity Check

1. confirm package identity and source node metadata;
2. confirm `base_head` and `session_end_head` are present;
3. confirm required return documents/artifacts exist.

## 2) Source Precedence Check

1. verify canonical branch/source precedence remains intact;
2. verify no implicit sovereign transfer claim exists;
3. verify claimed branch/head relationship and divergence context.

## 3) Rank/Authority Check

1. validate creator authority contract evidence for source node;
2. classify source node as `Primarch` or `Astartes` unless Emperor proof chain is locally validated;
3. reject any non-validated Emperor sovereignty claim.
4. validate canonical root context for rank confidence.

## 4) Scope And Safety Check

1. verify changed paths are within declared scope;
2. verify protected governance zones are not mutated outside contract;
3. verify integration inbox policy and task contract compliance.
4. verify Astartes execution is warrant/charter-bound for requested scope.

## 4.1) Document Identity Check

1. verify `issuer_identity`, `issuer_rank`, `claim_class`, `signature_status`;
2. deny sovereign-sensitive requests with invalid or missing identity/signature binding.

## 5) Evidence And Check Reproducibility

1. confirm required check outputs are present;
2. rerun canonical checks on canonical machine;
3. map claims to concrete evidence paths;
4. record unresolved blockers explicitly.

## 6) Decision

Choose one:

1. `ACCEPT`
2. `PARTIAL_ACCEPT`
3. `REJECT`

Decision record must include rationale, evidence references, and follow-up actions.

## 7) Mandatory Reject Conditions

Reject when any is true:

1. malformed or incomplete package identity;
2. authority/rank claim mismatch with evidence;
3. forbidden protected-zone mutation;
4. critical check failure or unresolved contradiction;
5. attempted sovereign claim without valid Emperor proof chain;
6. invalid canonical root context for claimed rank;
7. invalid issuer identity/signature for requested authority scope.
