# Integration Inbox

## Purpose

Canonical intake path for external helper block deliveries.

## Folder Roles

- `integration/inbox/` - newly dropped handoff packages.
- `integration/review_queue/` - review artifacts generated by integration audit flow.
- `integration/accepted/` - packages classified as `ACCEPT_CANDIDATE`.
- `integration/rejected/` - packages classified as `REJECT`.
- `integration/quarantine/` - packages classified as `QUARANTINE`.

## State Transitions

- helper nodes may only drop packages into `integration/inbox/`.
- integration review flow generates review artifacts in `integration/review_queue/`.
- only canonical creator-authority machine may finalize movement into accepted/rejected/quarantine states.

## Acceptance Semantics

- `ACCEPT_CANDIDATE` means candidate package passed integration audit contract.
- `REJECT` means contract/scope failure.
- `QUARANTINE` means malformed or suspicious package requiring manual inspection.
