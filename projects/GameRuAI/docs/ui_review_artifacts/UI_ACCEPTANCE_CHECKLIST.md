# UI Acceptance Checklist

## Blocking Checks
- [ ] No `FAIL` in `ui_validation_summary.json`.
- [ ] No critical findings remain unresolved.
- [ ] All required CTA controls visible for supported presets.
- [ ] No clipping/overflow on primary action controls.
- [ ] Screenshot manifest is complete and machine-readable.

## Manual Review Checks
- [ ] Action hierarchy is understandable without onboarding.
- [ ] Localization strings remain readable in dense panels.
- [ ] Resize behavior does not hide context-critical widgets.
- [ ] Companion/reports/voice tabs remain navigable with realistic data.
